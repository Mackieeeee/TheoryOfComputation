Conversion from NFA to DFA


An NFA can have zero, one or more than one move from a given state on a given input symbol. An NFA can also have NULL moves (moves without input symbol). On the other hand, DFA has one and only one move from a given state on a given input symbol.


Steps for converting NFA to DFA:


Step 1: Convert the given NFA to its equivalent transition table
To convert the NFA to its corresponding transition table, we must first list all of the states, input symbols, and transition rules. The transition rules are expressed as a matrix with rows representing the current state, columns representing the input symbol, and cells representing the next state.


Step 2: Create the DFA’s start stateThe collection of all potential starting states in the NFA is the DFA's start state. The "epsilon closure" of the NFA's start state is referred to as this set. The set of all states that may be reached from the start state by following epsilon () transitions is known as the epsilon closure.


Step 3: Create the DFA’s transition tableThe DFA's transition table is similar to the NFA's transition table, except that the rows and columns reflect groups of states rather than individual states. The epsilon closure of the set of states obtained by executing the transition rules in the NFA's transition table is stored in the relevant cell in the transition table for each input symbol.


Step 4: Create the DFA’s final statesThe final states of the DFA are the groups of states that contain at least one final state from the NFA.


Step 5: Simplify the DFAThe DFA generated in the preceding steps may have redundant states and transitions. We can use the following strategies to simplify the DFA:


  • Remove unreachable states: States that are inaccessible from the starting point can be eliminated from the DFA.

  • Remove dead states: States that are incapable of leading to a final state may be deleted from the DFA.

  • Merge equivalent states: States with the same transition rules for all input symbols can be combined into one.

Step 6: Repeat steps 3-5 until no further simplification is possible
We repeat steps 3-5 until no further simplification of the DFA is achievable. The obtained final DFA is the reduced DFA equivalent to the specified NFA.
